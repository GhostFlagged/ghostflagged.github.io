Recon-Dashboard

Published on November 2025

← Back to Home

Recon-Dashboard is a lightweight orchestration layer that chains together my favorite reconnaissance utilities. The goal is to automate the repetitive discovery work and leave me with a single, neatly organized folder per target containing everything I need for deeper analysis.

How It Works

The pipeline processes domains through a sequential workflow of reconnaissance tools. It reads target domains from targets.json and executes each module in order, with downstream tools consuming the previous tool’s output.

  1. Subfinder discovers subdomains for every target.
  2. httpx probes discovered subdomains to identify live hosts.
  3. Katana crawls each live host to enumerate endpoints and URLs.
  4. gau extracts JavaScript file URLs from Katana’s output.
  5. fallparams pulls query parameters from the Katana crawl data.

All results are saved inside the Targets/ directory and organized by domain with file names like {domain}.subfinder, {domain}.httpx, or {domain}.katana. A built-in skip mechanism checks whether a tool has already produced valid data; if so, that step is skipped to keep reruns fast.

Supported Modules

  • subfinder: High-signal subdomain discovery with passive + API sources.
  • httpx: Live host detection including response codes, titles, CDN tags, and tech fingerprints.
  • katana: Modern crawler for endpoints, URLs, and script references.
  • gau: Quick JavaScript URL extraction from historical data providers.
  • fallparams: Parameter discovery to prime future fuzzing runs.

Why I Built It

The dashboard mindset keeps me honest about coverage. Instead of juggling loose text files, I get predictable folders, easy diffing between runs, and clear signals when a module needs attention. The skip logic also means I can pause and resume large recon batches without wasting API credits or time.
